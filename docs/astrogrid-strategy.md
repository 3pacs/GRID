# AstroGrid Strategy

## Separation
AstroGrid is not GRID.

It shares the understructure.
Not the mask.

Shared:
- database
- selected models
- selected prediction outputs

Separate:
- brand
- voice
- interface
- release lane
- commercial face

## Nature
GRID computes.

AstroGrid divines.

GRID measures pressure.

AstroGrid reads pattern.

## Commerce
AstroGrid may sell:
- astrology-driven financial analysis
- horoscopes
- readings
- symbolic artifacts
- NFTs
- compute
- agent access

Money enters through signal, meaning, and access.

## Product Form
AstroGrid should hold:
- sky state
- ephemeris
- events
- user-selected belief lenses
- readings
- symbolic overlays
- financial timing signals
- separate cultural engines
- a final merged oracle layer

Not as a broker terminal.

As a ritual instrument with teeth.

## Tone
Cryptic.
Cold.
Exact.
Charged.

Never:
- wellness mush
- fintech drag
- fake priesthood
- word-sprawl

## First Rite
Ship:
- standalone shell
- separate runtime
- demo mode
- live celestial data
- own navigation
- own identity

Delay:
- payment labyrinths
- NFT machinery
- public sprawl
- side quests

## Rooms
- Sky
- Ephemeris
- Lenses
- Events
- Engines
- Seer
- Readings
- Signals
- Vault
- Logs
- Settings

Prototype map:
- `orrery` -> `Sky`
- `lunar` -> `Sky` or `Events`
- `ephemeris` -> `Ephemeris`
- `timeline` -> `Events`
- `narrative` -> `Readings`
- `correlations` -> `Signals`
- `settings` -> `Settings`

`Vault` is reserved for paid artifacts, drops, NFTs, or offerings.
`Lenses` controls which traditions are present, absent, or intersecting.
`Engines` holds the separate traditions.
`Seer` holds the merged voice.
`Logs` holds the ledgers.

## Shared Data Law
Same database.
Different temple.

- namespace AstroGrid data
- keep contracts narrow
- no hidden dependence on GRID internals
- AstroGrid must survive if GRID overlays vanish

## Prediction Law
AstroGrid may borrow from GRID.

It must not look borrowed.

Financial analysis is one chamber.
Not the whole cathedral.

AstroGrid predictions should arise from:
- computed celestial state
- separate cultural readings
- logged historical outcomes
- GRID world-state overlays
- user-selected lens rules

No forced syncretism.

## Branch Law
Use a separate branch.

Suggested:
- `codex/astrogrid-standalone`

If another agent occupies `astrogrid/src/*`:
- work docs
- work contracts
- work planning
- do not cross the circle

## Market Signal
AstroGrid is for people who want:
- celestial precision
- symbolic depth
- timing
- charged narrative
- usable omen

Avoid:
- “AI astrology”
- “Bloomberg for witches”
- certainty theater

## Final Shape
AstroGrid should feel like a sealed chamber where sky, signal, and price begin to rhyme.

Not one voice from the start.

Many voices.
Then the Seer.

And the user controls which voices enter the room.
